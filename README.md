---

🔑 Permission System

Implemented a permission system using Bitwise Operators ("|" and "&").

Each user has a permission mask that determines which system features they can access.

Users without the required permissions cannot open restricted screens.

---

💰 Transactions

- Deposit
- Withdraw
- Total Balances

- Currency Calculator
- Return to Main Menu

Displays the complete login history of system users.

---

🏗 Architecture

The project follows a 2-Tier Architecture, separating the Presentation Layer from the Business Logic.
File handling is integrated into the Business Layer rather than being isolated in a dedicated Data Access Layer.

- Presentation Layer
- Business Logic Layer & Data Layer
  
This separation makes the project easier to maintain, extend, and understand.

🎯 Project Goals

The primary goal of this project was to apply Object-Oriented Programming concepts in a real-world application
while improving software architectureand code organization skills.

It became one of the most valuable projects in my learning journey because it
transformed a procedural application into a fully object-oriented system with cleaner architecture and significantly better maintainability.
